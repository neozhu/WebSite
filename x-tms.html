X-TMS

方案介绍

运输管理系统（x-TMS）以物流运输管理为核心，统筹货主、收货人、运输企业、司机以及第三方物流企业等各环节运输信息，应用物联网、移动互联网等技术，实现物流运输全过程的协同化管理和可视化监控，提高物流运输过程工作效率和资源利用率，降低物流成本。 运输管理系统（TMS），具备车辆、线路、承运商、公司网点等各运输节点信息管理能力，通过任务协同完成从订单、调度、运输计划、运输执行、在途监控和运单结算的物流运输全流程可视化管理，通过对流程和成本的双重管控，实现高效率、低成本的运输服务。

IT 部门现在比以往任何时候都更需要实现事半功倍的效果。借助系统管理工具和解决方案，来帮助实现平衡并从中受益。


x-TMS是由我们自主研发的一款专门针对中、小物流企业的专业运输管理系统，由多位专业物流管理经验丰富的系统工程师并收集多家不同种类物流企业管理经验打造而成。 系统主体思路以用户体验为中心，我们的技术开发人员、架构设计人员都有十多年物流行业管理经验，系统特色体现在操作简单、极易上手、预警状态一目了然但不失丰富的功能及扩展性， 为小型企业制定业务流程也为大型企业预留各种功能扩充支持。

系统架构

我们采用目前最流程的ASP.NET MVC5 + N-Tier架构 + UnitOfWork设计模 优势：

  • 1,可伸缩性。这是由于多层的功能和低耦合性所决定的。比如，由于没有其他层的耦合，数据层可以扩大数据库集群，web客户端可以通过负载平衡器扩大而不影响其他层，Windows服务器可以轻松进行集群通过负载平衡和故障转移。
  • 2,可安全性。更好和更安全的控制整个系统，我们可以对每个层执行不同的安全策略，比如，业务层和数据层通常比表示层需要更高的安全级别，我们可以把这两高安全层放在防火墙后面进行保护。 3,可容错性。比如，在不影响其他层的情况下，数据库在数据层中可以为故障转移，进行负载平衡集群。
  • 4,可独立性。在不影响其他层情况下，可以进行独立升级和改变。在面向对象世界里，接口依赖实现可以把所有层解耦的非常好，那么就会导致如果其中某一层改变了，对于其他层是影响是非常小甚至是不影响。接口依赖意味着层跟层之间仅仅通过接口来互相通信，一层依赖于另一层的接口，而不是内部类来通信。当然，在改变整个系统时候，层的依赖性影响到的只是他低层次的实现，进而把其中副作用的影响降到最低。比如，如果接口不改变，在不影响整个系统下，我们可以更改或替换这个接口所实现的层。
  • 5,可便捷性。更便捷和更高效的开发环境，解耦主要是通过软件组件组合来实现某一模块，这样软件开发是非常便捷和高效的。可以把每一层要实现的功能单独分配给不同的开发组，只需通过接口来互相通信，每个层又可以自己做单元测试，到最后完成时组合起来就变成一个完整的系统。
  • 6,可维护性,可扩展性。由于对业务开发是以组件式来的，对于新功能的添加和删除是非常方便的。
  • 7,可重用性。由于是高内聚和低耦合，通用的功能和代码可以重复使用，也可以被其他更多的应用程序调用。

x-TMS支持目前所有主流的数据库产品，SQL Server（推荐使用），Oracle，MySQL

Smart Code Engine是我们自主研发的并且开源的一套居于ASP.NET MVC5(将来会支持 ASP.NET core)快速开发平台。集成系统必要的基础功能外，还有强大的代码自动生成功能, 保证开发人员安一致的规范和约束完成任务，保证开发效率和产品质量。

主要功能模块

x-TMS能支持第三方物流、零担运输、干线运输、城市配送、milk-run等业务，为企业提供方便快捷的运输管理服务

系统优势

  • 支持多种运输业务：全面支持零担专线、整车直发、城市配送、多式联运、循环取货、集或配送等多种运输业务，适用多种物流企业；
  • 全程成本利润管控：成本利润核算贯穿物流订单执行全流程，以订单为单元从报价的利润预估到每段运输过程的成本核算，为物流企业管理者提供每单利润监控，并从线路、司机、车辆各角度提供经营报表，帮助物流企业管理者进行运输经营分析；
  • 可视智能物流调度： 通过科学的调度机制，根据客户、接送货地址、货物属性、任务紧急程度等多维度数据进行合理且灵活的车辆调度，全面提高车辆调度准确性和调度效率，提升企业车辆利用率；
  • 业务流程可视：对运输过程中的车辆状态、运行轨迹、实时位置进行监控，并根据线路安排，进行节点控制实现签到及路线偏离预警，全新的电子看板可以全局的掌握所有信息，让运输状态一目了然。
  • 协同办公： 利用消息和任务推送的方式来推动工作的进展，使用移动APP，微信，在线聊天等功能保证随时随地保持沟通。
  • 开放的系统接口： 提供B2B/EDI的多种系统对接方案，方式于第三方系统比如WMS，ERP，CRM等实现EDI对接，减少人工操作，提高运行效率 。
